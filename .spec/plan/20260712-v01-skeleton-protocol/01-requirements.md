## 1. 背景与目标

### 1.1 背景

Rcode 是一个本地运行的 AI Agent 系统，参考 Claude Code 核心架构实现。v0.1 需要建立项目骨架和基础通信协议，为后续功能开发奠定基础。

### 1.2 目标

- 建立双进程架构（Core 守护进程 + CLI 客户端）
- 实现 JSON-RPC 2.0 + NDJSON 的 IPC 通信协议
- 实现基本的 ping/pong 命令验证通信链路
- 建立配置管理、日志系统等基础设施

## 2. 用户场景

### 2.1 目标用户

- 开发者：使用 Rcode 进行 AI Agent 开发
- 终端用户：通过 CLI 与 Agent 交互

### 2.2 使用场景

| 场景 | 用户 | 触发条件 | 期望结果 |
|------|------|----------|----------|
| 启动 Core | 开发者 | 运行 `rcode-core` | Core 守护进程启动并监听端口 |
| 测试连接 | 开发者 | 运行 `rcode ping` | 收到 pong 响应，显示 server、uptime、latency |
| 端口冲突 | 开发者 | 启动第二个 Core | 提示端口已被占用 |

## 3. 功能范围

### 3.1 做什么（In Scope）

- [x] 双进程架构（Core + CLI）
- [x] JSON-RPC 2.0 协议实现
- [x] TCP 传输层（NDJSON）
- [x] 配置管理（四级优先级）
- [x] ping/pong 命令
- [x] 端口占用检测
- [x] 优雅关闭（信号处理）
- [x] 单元测试和集成测试

### 3.2 不做什么（Out of Scope）

- Agent Loop（v0.2）
- 工具系统（v0.2-v0.3）
- 事件系统（v0.4）
- 会话管理（v0.6）
- 权限系统（v0.9）

## 4. 验收标准

| 编号 | 验收条件 | 优先级 |
|------|----------|--------|
| AC-01 | 运行 `rcode-core` → Core 启动并监听 127.0.0.1:7437 | P0 |

## 5. 非功能需求

### 5.1 性能

- ping 响应时间：≤ 10ms（本地）
- 支持并发连接：≥ 10 个客户端

### 5.2 安全

- 仅监听 localhost（127.0.0.1）
- 配置文件权限隔离

### 5.3 兼容性

- Python 3.12+
- Linux/macOS（Windows 待验证）
