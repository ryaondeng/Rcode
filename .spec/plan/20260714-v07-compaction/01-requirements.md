# v0.7 上下文压缩 — 需求文档

> Spec: `20260714-v07-compaction`
> 阶段：需求阐述
> 日期：2026-07-14
> 状态：待确认

## 1. 背景与目标

### 1.1 背景

v0.6 已实现会话管理，但长对话场景下 token 会溢出。需要实现上下文压缩，支持 token 管理。

### 1.2 目标

- 实现四层压缩管线（参考 learn-claude-code）
- 实现 tool_result 截断（始终生效）
- 实现 LLM 驱动的上下文压缩（可选）
- 保持 tool_use/tool_result 配对完整性

## 2. 用户场景

| 场景 | 用户 | 触发条件 | 期望结果 |
|------|------|----------|----------|
| 长对话 | 开发者 | 对话超过 token 限制 | 自动压缩上下文 |
| 手动压缩 | 开发者 | 主动触发压缩 | 生成摘要 |
| 工具结果过长 | 系统 | tool_result 超长 | 自动截断 |

## 3. 功能范围

### 3.1 做什么（In Scope）

- [ ] TokenBudget（tool_result 截断）
- [ ] Compactor（LLM 压缩）
- [ ] 四层压缩管线
- [ ] tool_use/tool_result 配对保护

### 3.2 不做什么（Out of Scope）

- 压缩可视化
- 压缩历史查询
- 自定义压缩策略
