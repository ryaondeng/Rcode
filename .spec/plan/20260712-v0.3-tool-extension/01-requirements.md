## 1. 背景与目标

### 1.1 背景

v0.2 已实现 Agent 最小闭环，但只有 bash 工具。需要扩展更多内置工具，提升 Agent 的文件操作能力，让 Agent 能够更好地理解和操作项目。

### 1.2 目标

- 添加文件操作工具（read_file、write_file、edit_file）
- 添加目录操作工具（list_dir、glob）
- 实现路径安全校验（safe_path）
- 保持工具原子化，每个工具只做一件事

## 2. 用户场景

### 2.1 目标用户

- 开发者：使用 Rcode 辅助编码和项目操作

### 2.2 使用场景

| 场景 | 用户 | 触发条件 | 期望结果 |
|------|------|----------|----------|
| 读取代码 | 开发者 | 需要查看文件内容 | Agent 读取并显示文件 |
| 创建文件 | 开发者 | 需要创建新文件 | Agent 创建文件并写入内容 |
| 编辑代码 | 开发者 | 需要修改现有文件 | Agent 精确替换指定文本 |
| 查找文件 | 开发者 | 需要查找特定类型的文件 | Agent 返回匹配的文件列表 |
| 了解结构 | 开发者 | 需要了解项目目录结构 | Agent 显示目录树 |

## 3. 功能范围

### 3.1 做什么（In Scope）

- [ ] `read_file` 工具：读取文件内容
- [ ] `write_file` 工具：写入文件
- [ ] `edit_file` 工具：精确文本替换
- [ ] `list_dir` 工具：列出目录内容
- [ ] `glob` 工具：文件模式匹配
- [ ] `safe_path` 路径安全校验
- [ ] WORKDIR 工作目录定义

### 3.2 不做什么（Out of Scope）

- 参数校验（Pydantic）- 留给后续版本
- 事件发布 - 属于 v0.4
- 权限检查 - 属于 v0.9
- 重试机制 - 属于 v0.10

## 5. 非功能需求

### 5.1 性能

- 文件读取限制：512KB
- 文件写入限制：1MB
- 目录列表限制：200 条目
- glob 结果限制：100 文件

### 5.2 安全

- 所有路径操作通过 safe_path() 校验
- 禁止路径逃逸到工作目录之外
- 禁止符号链接攻击
