## 1. 背景与目标

### 1.1 背景

目前配置散落在三处，加新字段容易错位：
- `AgentRunner.__init__` 里硬编码默认值
- `load_config` 简易 dict
- 命令行参数

| 问题 | 代码证据 | 影响 |
|------|----------|------|
| 配置散落三处 | Runner 构造函数 / load_config / CLI 参数 | 加新字段容易错位 |
| 无类型校验 | dict 直接使用 | 拼错字段不报错 |
| 无分节 | 所有配置扁平放 | 配置多了难管理 |
| 无来源追踪 | 不知道值从哪来 | 调试困难 |

### 1.2 目标

将配置统一到"四层优先级 + pydantic 校验 + 未知 key 硬拒"的配置模型。

- 四层优先级：默认值 → 全局 → 本地 → 环境变量/CLI
- 未知 key 硬拒：拼错字段名立即报错
- 配置查看命令：`rcode config show`

## 3. 功能范围

### 3.1 做什么（In Scope）

- [ ] pydantic 配置模型（四层分节）
- [ ] 配置加载器（四层优先级）
- [ ] 未知 key 硬拒
- [ ] 环境变量支持
- [ ] `rcode config show` 命令
- [ ] Runner 适配

### 3.2 不做什么（Out of Scope）

- 不做热加载（改配置后重启 Core）
- 不做加密敏感字段（未来）
