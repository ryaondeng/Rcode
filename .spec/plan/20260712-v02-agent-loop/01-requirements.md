# v0.2 Agent 最小闭环 — 需求文档

> Spec: `20260712-v02-agent-loop`
> 阶段：需求阐述
> 日期：2026-07-12
> 状态：待确认

## 1. 背景与目标

### 1.1 背景

v0.1 已完成骨架与协议，建立了双进程架构和 IPC 通信。现在需要实现 Agent 的核心能力：接收用户目标，调用 LLM 进行思考，执行工具调用，完成任务。

### 1.2 目标

- 实现最小可运行的 Agent Loop
- 实现 LLM 接口（Anthropic API）
- 实现工具系统（注册、调用）
- 实现至少一个内置工具（bash）
- 保持架构扩展性，便于后续版本扩展

## 2. 用户场景

### 2.1 目标用户

- 开发者：使用 Rcode 执行自动化任务

### 2.2 使用场景

| 场景 | 用户 | 触发条件 | 期望结果 |
|------|------|----------|----------|
| 执行任务 | 开发者 | `rcode run --goal "创建 hello.py"` | Agent 完成任务并返回结果 |
| 工具调用 | Agent | LLM 请求调用 bash 工具 | 工具执行并返回结果 |
| 任务完成 | Agent | LLM 判断任务完成 | 停止循环，返回最终答案 |

## 3. 功能范围

### 3.1 做什么（In Scope）

- [ ] Agent Loop（ReAct 循环）
- [ ] LLM Provider（Anthropic API）
- [ ] Tool System（注册表模式）
- [ ] ExecutionContext（执行上下文）
- [ ] AgentRunner（组装依赖）
- [ ] Bash Tool（内置工具）
- [ ] `rcode run` 命令

### 3.2 不做什么（Out of Scope）

- 流式输出（v0.4 事件流）
- 重试机制（v0.10 错误恢复）
- 权限系统（v0.9）
- 会话管理（v0.6）
- 上下文压缩（v0.7）
- 子 Agent（v0.12）

## 4. 验收标准

| 编号 | 验收条件 | 优先级 |
|------|----------|--------|
| AC-01 | 运行 `rcode run --goal "创建 hello.py"` → Agent 完成任务 | P0 |
| AC-02 | LLM 能够正确调用 bash 工具 | P0 |
| AC-03 | 工具结果能够正确回填到上下文 | P0 |
| AC-04 | Agent 能够判断任务完成并停止 | P0 |
| AC-05 | 步数超限时能够终止循环 | P1 |
| AC-06 | 单元测试覆盖率 ≥ 80% | P1 |

## 5. 非功能需求

### 5.1 性能

- LLM 调用超时：≤ 60s
- 单次任务最大步骤：≤ 20 步

### 5.2 安全

- Bash 工具执行有超时保护（30s）
- 输出截断防止过长（50KB）

### 5.3 兼容性

- Python 3.12+
- 支持 Anthropic API
