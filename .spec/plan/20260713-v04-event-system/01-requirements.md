# v0.4 事件流 — 需求文档

> Spec: `20260713-v04-event-system`
> 阶段：需求阐述
> 日期：2026-07-13
> 状态：待确认

## 1. 背景与目标

### 1.1 背景

v0.3 已完成工具扩展，但 Agent 执行过程没有实时外化。用户无法看到 Agent 的思考和工具调用过程。

### 1.2 目标

- 实现 EventBus 发布-订阅机制
- 定义完整的事件类型
- 实现事件持久化到 JSONL 文件
- 在 Agent Loop 中发布关键事件

## 2. 用户场景

| 场景 | 用户 | 触发条件 | 期望结果 |
|------|------|----------|----------|
| 实时监控 | 开发者 | 执行 Agent 任务 | 看到 Agent 思考和工具调用 |
| 调试分析 | 开发者 | Agent 行为异常 | 回放事件流定位问题 |
| 审计追踪 | 管理员 | 需要操作记录 | 查看完整执行历史 |

## 3. 功能范围

### 3.1 做什么（In Scope）

- [ ] EventBus 发布-订阅机制
- [ ] 8 种事件类型定义
- [ ] EventWriter 事件持久化
- [ ] Agent Loop 事件发布

### 3.2 不做什么（Out of Scope）

- Trace 追踪（v0.5）
- TUI 事件显示（后续版本）
- 事件过滤和查询（后续版本）
